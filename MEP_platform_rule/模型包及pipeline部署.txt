1.为什么需要使用到MEP平台？
当在内部Notebook/调试平台的notebook上调通pipeline后，由于我们需要跑大批量的数据进行清洗，该平台notebook的算力是无法满足的，它只能作为调试的平台，我们需要使用MEP平台上的310集群才能够跑批量数据。此外，由于MEP上是310集群环境，而我们用于调试的是v100卡，所以还需要在昇腾310服务器上进行环境适配。

所以，构建一个数据处理pipeline的完整闭环流程应该是：

算法设计

本地/内部Notebook调通

昇腾服务器310环境适配

MEP组件包以及模型包打包

部署到MEP

3. MEP平台部署准备件
我们来梳理一下流程，假设你的原始代码是在本地/内部Notebook平台的notebook上调通了，之后的步骤包括：

1.昇腾310环境适配：由于v100和310的基础架构不一样，我们需要在一台310昇腾服务器上适配好环境，跑通代码。

2.模型包：我们的原代码中，若用到多个模型进行推理，必定有对应的模型参数文件，需要单独打包为模型包。模型包的作用就是放置模型参数文件。

3.代码组件包：使用异步场景组件包,这个组件包里是要重构代码，CustomerModel.load()函数里加载模型以及一些耗时的初始化操作。框架会调用calc()方法并传递业务的请求数据req_Data，在calc()中需支持对业务数据的接收、业务逻辑处理。

install_packages_from_folder函数如下：

import os
import subprocess
import sys
def install_packages_from_folder(folder_path):
    # 获取 resource 文件夹中所有包的完整路径
    package_paths = [os.path.join(folder_path, f) for f in os.listdir(folder_path) if f.endswith('.whl') or f.endswith('.tar.gz')]
    
    if package_paths:
        # 使用 pip 一次性安装所有包
        print("Installing packages from resource folder...")
        subprocess.check_call([sys.executable, '-m', 'pip', 'install', *package_paths])
    else:
        print("No packages found in the folder.")

这张图展示的是一个压缩模型包解压后的目录结构示意图。
从上到下看：

* 最上面是一个压缩文件：xx_xx.zip
* 解压后得到一个目录：modelDir
* modelDir 下面包含 3 个子目录：
    * meta
    * model
    * data
* 其中 meta 目录下面还有一个文件：type.mf

可以把它理解成：

1. zip 压缩包是整体交付文件
2. 解压后形成 模型主目录
3. 主目录里通常分成：
    * meta：保存元信息、配置说明
    * model：保存模型本体
    * data：保存相关数据
4. type.mf 很可能是一个类型/格式说明文件

这是一张层级结构图，重点不是具体内容，而是说明这个模型包内部文件夹之间的组织关系。


注意

modelDir名称固定，不要修改，修改后会导致后续模型解压失败。

data目录下为训练过程中产生的模型评估数据和模型请求样例sample.json，可选；

model目录下为真正的模型文件，比如PMML文件等，必选。

meta目录下固定为type.mf，标识模型的算法类型等信息，必选。

模型文件命名格式：业务_场景_算法_备注（可选），譬如appstore_game_spark，表示应用市场游戏场景spark模型。appstore_ads_dcn_diversty表示应用市场付费推广DCN算法Diversty策略模型。

# 构建请求头
def calc_sign(data, sign_key, query_param, path, method, app_id):
        """
        云服务推理模型提前的签名算法，未进行过变更
        :param data: 数据体
        :param sign_key: 签名key
        :param query_param: 请求参数
        :param path: 请求路径
        :param method: 请求方式
        :param app_id: appId
        :return: 时间戳,签名
        """
        payload = ""
        if data:
            payload += data
        query_str = ""
        path_str = ""
        for k, v in query_param.items():
            query_str += "{}={}&".format(k, v)
        query_str = query_str[:-1]
        for v in path:
            path_str += "/{}".format(v)
        timestamp = int(time.time() * 1000)
        sign_str = "{}&{}&{}&{}&appid={}&timestamp={}".format(method, path_str, query_str, payload, app_id, timestamp)
        sign_key = sign_key.encode('utf-8')
        sign_str = sign_str.encode('utf-8')
        sign = base64.b64encode(hmac.new(sign_key, sign_str, digestmod=hashlib.sha256).digest()).decode("utf-8")
        return timestamp, sign


# 发送请求，在这里构建多线程
def fetch_url(MEP_URL,headers,body,latch):
    try:
        start_time = time.time()
        response = requests.request('POST', MEP_URL, headers=headers, data=body)
        elapsed_time = time.time() - start_time
        resp_json = response.status_code
    finally:
        latch.count_down()

    return resp_json, elapsed_time


class CountDownLatch:
    def __init__(self, count):
        self.count = count
        self.event = threading.Event()

    def count_down(self):
        self.count -= 1
        if self.count == 0:
            self.event.set()

    def latch_await(self, timeout=None):
        self.event.wait(timeout)


def test_concurrency(num_requests,NUM_REQUESTS,MEP_URL, headers, body):
    # 使用线程池执行并发请求
    latch = CountDownLatch(num_requests)
    executor = ThreadPoolExecutor(max_workers=num_requests) 

    # 提交所有请求任务

    future_to_url = {executor.submit(fetch_url, MEP_URL, headers, body,latch): MEP_URL for _ in range(num_requests)}

    executor.shutdown(wait=True)
    latch.latch_await(num_requests)

    # 等待所有请求完成，并打印结果
    for future in as_completed(future_to_url):
        status_code, elapsed_time = future.result()
        print(f"URL: {MEP_URL}, Status Code: {status_code}, Time: {elapsed_time:.4f} seconds")
