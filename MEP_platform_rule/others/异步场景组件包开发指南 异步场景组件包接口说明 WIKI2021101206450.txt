3.1 接口描述
业务调用MEP的该接口，可执行对应场景的模型，根据预定的路由规则分发到不同的模型服务，并返回对应的模型执行结果。下面以图像处理为例，介绍异步处理的流程：

采用异步接口，模型服务与业务服务会通过SFS共享图片存储；业务请求中传入一组图片存储位置信息、相应的处理要求，以及处理后结果文件的存放位置；模型服务接受到请求后放入异步队列后台进行处理；业务服务器定期过来查询任务处理情况，待任务处理完成后，业务服务读取结果文件内容并进行后续处理。如图所示：

POST /service

参数定义

参数列表

类型

M/O

字段描述

version

String{1,64}

M

服务协议版本号，目前值为1.0

meta

JSONObject

M

请求元数据，由框架约定，JSON结构体

data

JSONObject

M

请求具体数据，JSON结构体 ,各个业务场景定制

 

meta对象

字段名

类型

M/O

描述

bId

String{1,128}

M

业务标识，例如某浏览器类业务、某典型行业应用（示例）

flowId

String{1,128}

M

业务场景，比如裁剪图片，图片语义理解

subId

String{1,128}

O

子场景标识，可选

uuId

String{1,128}

O

请求唯一标识，可以使用uuid、时间戳

traceId

String{1,128}

O

日志跟踪唯一标识，用于调试问题，如果请求中携带了traceID会把各个部件关联的日志(包含debug级别)输出到统一的文件

消息样例

以下接口样例都是业务侧——>消息网关/服务端的参数说明

 

POST /service

#创建任务场景
Content Type:
application/json;charset=UTF-8

{
    "version": "1.2",
    "meta": {
        "bId": "businessId",
        "flowId": "imageprocess",
        "uuId": "1502346942471"
    },
    "data": {
        "taskId": "100002455",
        "action": "create",
        "basePath": "/opt/business/businessId /100002455",
        "fileInfo": [
            {
                "sourceImage": "image1.png",
                "sourcePath": "/opt/business/businessId/100002455/sourcepath",
                "generatePath": "/opt/business/businessId/100002455/generatePath",
                "processSpec": []
            }
        ]
    }
}


#查询任务场景
{
    "version": "1.2",
    "meta": {
        "bId": "businessId",
        "flowId": "imageprocess",
        "uuId": "1502346942471"
    },
    "data": {
        "taskId": "100002455",
        "action": "query",
        "basePath": "/opt/business/businessId /100002455"
    }
}

响应码取值返回及定义

响应码

场景

描述

0

创建任务/查询任务

1、  如请求为创建任务，返回0表示任务创建成功

2、  如请求为查询任务，返回0表示任务执行完成并成功

1

创建任务

该响应码用于创建任务，返回1表示系统忙需要重试

2

查询任务

该响应码用于查询任务，返回2表示任务正在处理中

3

创建任务/查询任务

1、如请求为创建任务，返回3表示任务创建结束，异常

2、如请求为查询任务，返回3表示任务查询结束，异常

4

查询任务

该响应码用于查询任务，返回4表示没有对应的任务

5

创建任务

该响应码用于创建任务，返回5表示所创建任务已存在，创建了重复任务
