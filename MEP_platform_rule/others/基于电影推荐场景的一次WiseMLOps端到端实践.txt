1. 背景介绍
案例选取的是推荐场景下，运用NeuMF模型，对MovieLens-20M进行训练、推理。

MovieLens-20M

MovieLens-20M是一个稳定的基准数据集。此数据集包含了138493个用户对27278个电影的20000263个评分和465564个标签。

用户为随机选取，每个用户只给出一个ID，且不涉及其他私人信息。

MovieLens-20M以csv格式存储，便于直接可视化分析。

文件列表：genome_scores.csv、genome-tags.csv、links.csv、movies.csv、ratings.csv、tags.csv。

movies.csv

MovieId+title+geners。以此表示电影ID、电影名称、电影流派/种类。其中电影流派具有多个标签，即可以表示电影的多个属性。



ratings.csv

userId+movieId+rating+timestamp。分别表示用户ID、电影ID、评分，以及截至时间戳。给出了用户对电影的评分列表。



tags.csv

userId+ movieId+tag+timestamp。分别表示用户ID、电影ID、用户对电影的标签、时间戳。给出了用户对电影的标签列表。



links.csv

moviesId+imdeId+tmdbId。IMDB为互联网电影资料库。tMDB为电影数据集。给出了电影ID和两个数据标记ID的对应关系。



genome_tags.csv

电影标签 DNA标记，唯一标识符。



genome_scores.csv

movieId+tagId+relevance。分别表示电影ID、电影标签ID、官方标签相关性。给出了电影的官方标签。



 

NeuMF模型

NeuMF模型基于神经网络的协同过滤框架NCF，通过隐式反馈解决个性化排名任务。在推荐系统中，点击、购买和观看等操作是常见的隐式反馈，易于收集并能一定程度上代表用户的偏好。

NeuMF模型的结构图如下所示：



 

GMF为广义矩阵分解，通过MF user向量和MF item向量之间的逐元素相乘，将生成向量映射到GMF层。

MLP为多层感知器，在多模态深度学习中被广泛应用。在MLP user向量和MLP item向量连接的基础上，增加隐藏层。对于感知层的激活函数，ReLU激活函数是较好的选择，因为相比Sigmoid和tanh，ReLU更具有生物合理性，并且被证明不会导致过饱和。

NeuMF模型结合了GMF的线性度和MLP的非线性度对用户-物品之间的潜在结构进行建模，通过权重在输出层将GMF和MLP预训练的向量连接起来。

     iv.  使用基于Apex的amp对训练过程进行混合精度加速

opt_level共有四个等级

O0相当于原始的单精度训练。

O1在大部分计算时采用半精度，但是所有的模型参数依然保持单精度，对于少数单精度较好的计算（如softmax）依然保持单精度。

O2相比于O1，将模型参数也变为半精度。

O3基本等于最开始实验的全半精度的运算

这里选择的是O2对模型训练进行加速。

keep_batchnorm_fp32	None	None	True	False
master_weights	False	None	True	False
loss_scale	1.0	“dynamic”	“dynamic”	1.0

模型训练时，前向传播过程中传入了user和item向量，经过GMF层和MLP层，通过权重将向量连接得到预测输出。

在得到输出后，首先与真实label相比较，其次通过amp.scale_loss 进行回传。在反向传播的过程中更新优化器的学习率。

    v.  验证时，验证数据集（即每个用户最后一次给电影评分的数据）为真实结果。除此之外，每个用户生成999个负样本（即与用户没有交集的movieId）。

将新生成的样本（user-item）放入模型，并运用topK（）为每个用户从1000个电影中推荐10个电影。

若真实交互电影在前10个电影中，则记为1，反之记为0。

将命中次数/用户总数得到命中率。

通过命中率决定训练是否停止。
